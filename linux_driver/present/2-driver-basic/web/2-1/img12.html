兩個 Kernel API 需要瞭解:

A) cdev_alloc()

B) cdev_init()

我們可以看到有兩個方法去初始化 struct cdev (字元驅動程式主體模型)
兩個方法都可以, 看設計者如何使用

1) 宣告一個 struct cdev 的指標變數 struct cdev *my_cdev;
    剛開始 "my_cdev" 當然是一個 "空指標" 所以我們需要去初始化
    這種狀況, 我們就使用 cdev_alloc() 去初始化"字元驅動程式主體模型"


2) 宣告一個 struct cdev 的變數 struct cdev my_cdev;
    my_cdev 已是一個佔有系統記憶體位址的變數, 但是我們仍需要初始化 "字元驅動程式主體模型"
    我們就使用 cdev_init() 去初始化"字元驅動程式主體模型"


以上兩種狀況就是如此, 看設計者如何使用.
