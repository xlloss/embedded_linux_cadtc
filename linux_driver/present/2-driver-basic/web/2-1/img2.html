1. user-land 會透過 "Device Name" 與底層的 driver 溝通.
"Device Name" 基本上會被建立於根目錄統下 "/dev" 的地方
例如 : i2c device driver 可以在根目錄下看到 /dev/i2c-0, /dev/i2c-1, /dev/i2c-2 ...


2. 我們可以在 /proc/devices 下面確認字元驅動程式是否已經向系統註冊


3. 字元驅動程式 需要注意兩種代碼 :
主代號 (major member)
次代號 (minor member)


4. 例如:我們在 user-land 使用 file write API 去控制 device driver
