1. platform driver model 如投影片所示.


2. 裡面為 function point. 需要開發者去實做 platform device 相關的動作.


3. 其中相對較重要:
   a) probe :
     當 platform driver 被系統載入後, probe() 會被系統呼叫.

     因此同常我們會在 probe() 裡做 "初始化" 硬體 IP 的動作.
     例如 記憶體映射, IRQ 註冊, initialize Hardware ...

   a) remove :
     當 platform driver 被系統卸載, remove() 會被系統呼叫.

     因此同常我們會在 remove() 裡做資源釋放的動作.
     例如 記憶體歸還給系統.
