1. platform device 需要靜態宣告, 它無法被動態測.
    a）靜態宣告即是在編譯之前就要事先設定 (depend DeviceTree, static platform device setting).
     for example : 一個 I2C bus --> s3c2410-i2c.c


    b）動態偵測即是在 kernel 執行時透過某些方法載入
     for example : 一個 USB ethnet card
     usb bus 會去掃描掛在 usbhub 下 所有的 device. usb bus 找到一個裝置.
     它會去讀取裝置的 [PID VID]. 透過 [PID VID] 再去載入驅動程式.
     不需要做事先的 devicetree 的設定.
